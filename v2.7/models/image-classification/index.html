    Image species classification¶

    zamba ships with two models for classifying the species in camera trap images:

    • lila.science (default): a ConvNextV2 model trained on 178 species and groups from around the world.
    • speciesnet: a conversion of Google's SpeciesNet classifier with a very large global taxonomy of 2,000+ classes.

    Both models support training, fine-tuning, and inference, and both use MegaDetector to crop images to the animal before classification. To choose a model at prediction or training time, pass it with --model (see the classifying images and training on images tutorials).

    The full default configuration is available on Github.

    The default configuration will use megadetector to identify bounding boxes for animals and then use the lila.science model to identify the species in the bounding box.

    It will generate a CSV file with predicted species probabilities for each bounding box.

    speciesnet model¶

    Architecture¶

    The speciesnet model is a zamba-compatible conversion of Google's SpeciesNet classifier (model card on Kaggle). It uses an EfficientNetV2-M backbone and accepts 480x480 images as input. Like lila.science, it is an image-native classifier that runs on the cropped region around an animal.

    zamba uses the always_crop variant of SpeciesNet, which expects images already cropped to the animal's bounding box. Google also publishes a full_image variant that classifies the whole frame; zamba does not use it.

    Training data¶

    SpeciesNet was trained by Google on a geographically diverse dataset of over 65 million camera trap images, combining curated images contributed by the Wildlife Insights community with images from publicly available repositories. zamba uses the resulting classifier as-is (no additional zamba training); to adapt it to your own labels, see the training on images tutorial.

    Full details of the model and training are in the accompanying paper:

    Gadot T, Istrate Ș, Kim H, Morris D, Beery S, Birch T, Ahumada J. To crop or not to crop: Comparing whole-image and cropped classification on a large dataset of camera trap images. IET Computer Vision. 2024;18(8):1193–208.

    Performance and scope¶

    The classifier produces a probability for each of the 2,498 taxonomy labels; for any image, the most useful signal is typically the top-ranked label (or top-5).

    zamba runs the classifier, not the full SpeciesNet ensemble

    Google's SpeciesNet is usually deployed as an ensemble that combines the classifier with MegaDetector and optional geographic filtering ("geofencing"), plus taxonomic "roll-up" heuristics, to commit to a single label per image. zamba's speciesnet runs the classifier only on MegaDetector crops: it does not apply geofencing or the ensemble roll-up logic, so the outputs are the raw classifier probabilities over the full taxonomy. If you need geofencing or the ensemble behavior, run Google's SpeciesNet directly.

    Default configuration¶

    The full default configuration is available on Github.

    The default configuration uses MegaDetector to identify bounding boxes for animals and then runs the speciesnet model on each cropped box. Unlike lila.science, speciesnet resizes crops to 480x480 with bicubic interpolation and does not apply normalization; zamba selects this preprocessing automatically based on the model, so you do not need to configure it manually.

    To use speciesnet instead of the default lila.science model, pass it with --model:

    $ zamba image predict --data-dir example_images/ --model speciesnet
