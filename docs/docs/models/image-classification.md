# Image species classification

`zamba` ships with two models for classifying the species in camera trap **images**:

* **[`lila.science`](#lila.science)** (default): a ConvNextV2 model trained on 178 species and groups from around the world.
* **[`speciesnet`](#speciesnet)**: a conversion of Google's SpeciesNet classifier with a very large global taxonomy of 2,000+ classes.

Both models support training, fine-tuning, and inference, and both use [MegaDetector](megadetector.md) to crop images to the animal before classification. To choose a model at prediction or training time, pass it with `--model` (see the [classifying images](../images-predict-tutorial.md) and [training on images](../images-train-tutorial.md) tutorials).

The `speciesnet` model recognizes a much larger global taxonomy of **2,498 classes**, spanning individual species, higher-level taxa (e.g. `felidae`, `mammalia`), and the non-animal labels `blank` and `vehicle`. The labels break down by broad group as:

| Group | Labels |
| ------------------------------------------------------------------ | -----: |
| Birds (`aves`)                                                     |  1,357 |
| Mammals (`mammalia`)                                              |  1,069 |
| Reptiles (`reptilia`)                                             |     58 |
| Amphibians (`amphibia`)                                           |      4 |
| Invertebrates (`arachnida`, `insecta`, `diplopoda`, `gastropoda`, `malacostraca`) | 8 |
| Non-animal (`blank`, `vehicle`)                                  |      2 |

By taxonomic rank, there are 2,066 species-level labels plus 283 genus, 108 family, 30 order, and 9 class-level groups, drawn from 60 orders and 274 families. The list below shows the common name with the scientific name or taxon in parentheses.

## `lila.science` model

### Architecture

The `lila.science` model is a global model with a ConvNextV2 base size (87.7M parameters) backbone accepting 224x224 images as input.

## `speciesnet` model

### Architecture

The `speciesnet` model is a `zamba`-compatible conversion of [Google's SpeciesNet](https://github.com/google/cameratrapai) classifier ([model card on Kaggle](https://www.kaggle.com/models/google/speciesnet)). It uses an [EfficientNetV2-M](https://arxiv.org/abs/2104.00298) backbone and accepts 480x480 images as input. Like `lila.science`, it is an image-native classifier that runs on the cropped region around an animal.

`zamba` uses the **`always_crop`** variant of SpeciesNet, which expects images already cropped to the animal's bounding box. Google also publishes a `full_image` variant that classifies the whole frame; `zamba` does not use it.

### Training data

SpeciesNet was trained by Google on a geographically diverse dataset of **over 65 million camera trap images**, combining curated images contributed by the [Wildlife Insights](https://www.wildlifeinsights.org/) community with images from publicly available repositories. `zamba` uses the resulting classifier as-is (no additional `zamba` training); to adapt it to your own labels, see the [training on images](../images-train-tutorial.md) tutorial.

Full details of the model and training are in the accompanying paper:

> Gadot T, Istrate Ș, Kim H, Morris D, Beery S, Birch T, Ahumada J. [To crop or not to crop: Comparing whole-image and cropped classification on a large dataset of camera trap images](https://doi.org/10.1049/cvi2.12318). *IET Computer Vision*. 2024;18(8):1193–208.

### Performance and scope

The classifier produces a probability for each of the 2,498 taxonomy labels; for any image, the most useful signal is typically the top-ranked label (or top-5).

!!! tip "`zamba` runs the classifier, not the full SpeciesNet ensemble"
    Google's SpeciesNet is usually deployed as an *ensemble* that combines the classifier with MegaDetector and optional **geographic filtering ("geofencing")**, plus taxonomic "roll-up" heuristics, to commit to a single label per image. `zamba`'s `speciesnet` runs the **classifier only** on MegaDetector crops: it does not apply geofencing or the ensemble roll-up logic, so the outputs are the raw classifier probabilities over the full taxonomy. If you need geofencing or the ensemble behavior, run [Google's SpeciesNet](https://github.com/google/cameratrapai) directly.

### Default configuration

The full default configuration is available on [Github](https://github.com/drivendataorg/zamba/blob/master/zamba/models/official_models/speciesnet/config.yaml).

The default configuration uses MegaDetector to identify bounding boxes for animals and then runs the `speciesnet` model on each cropped box. Unlike `lila.science`, `speciesnet` resizes crops to 480x480 with bicubic interpolation and does **not** apply normalization; `zamba` selects this preprocessing automatically based on the model, so you do not need to configure it manually.

To use `speciesnet` instead of the default `lila.science` model, pass it with `--model`:

```console
$ zamba image predict --data-dir example_images/ --model speciesnet
```
